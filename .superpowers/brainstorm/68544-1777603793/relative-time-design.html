相对时间轴设计

多版本曲线从0秒对齐，便于对比同一阶段性能

时间轴设计核心

❌ 传统绝对时间（不推荐）
- 版本1: 14:20:00 ~ 14:30:00
- 版本2: 14:45:00 ~ 14:55:00
- 版本3: 15:10:00 ~ 15:20:00

问题：
- 时间不对齐，难以对比
- 曲线起始点不同
- 无法对比同一阶段性能
✅ 相对时间（推荐）
- 所有版本从0秒开始
- 曲线对齐显示
- 便于对比同一阶段

优势：
- 直观对比启动阶段
- 对比稳定运行阶段
- 标签保存相对时间点

Tooltip 显示内容

版本1
相对时间: 30秒
实际时间: 2026-05-01 14:20:30
系统CPU: 45.2%
进程CPU: 26.9%
版本2
相对时间: 30秒
实际时间: 2026-05-01 14:45:30
系统CPU: 38.5%
进程CPU: 21.1%
设计要点：
- Tooltip 同时显示相对时间（便于定位）和实际时间（便于追溯）
- 实际时间显示完整年月日时分秒
- 多版本时，各版本 Tooltip 并排显示

标签保存（相对时间）

打标签流程：
  1. 点击曲线上某个时间点（相对时间，如 30秒）
  2. 弹出配置框：标签名称、时间长度、区间类型
  3. 确认添加，保存相对时间点
标签数据结构： 
{
  "collect_id": "uuid",
  "name": "发起共享",
  "start_relative_time": 30,  // 相对时间（秒）
  "duration": 60,             // 时间长度（秒）
  "type": "peak"              // peak/mean
}
为什么用相对时间：
  • 适配不同采集时间（版本1采集10分钟，版本2采集15分钟）
  • 下次打开该版本时自动加载标签
  • 多版本对比时，标签可智能合并
最终方案确认：
  • 时间轴：相对时间（从0秒开始）
  • 曲线：多版本从0秒对齐显示
  • Tooltip：相对时间 + 完整实际时间
  • 标签：保存相对时间点，按版本保存
