# Worker 性能监控页面设计文档

> 创建日期: 2026-05-01
> 作者: Claude + 用户

## 一、功能定位

### 核心定位
- **一级菜单**：独立的性能监控模块
- **目标设备**：Windows 设备（一期）
- **用途**：实时调试 + 历史数据追溯

### 数据采集维度

#### 系统级别指标
| 指标 | 位置 | 说明 |
|------|------|------|
| CPU 使用率 | 左侧主曲线图 | 系统曲线 + 目标进程曲线叠加，有 TOP10 Tooltip |
| 提交内存 | 左侧主曲线图 | 系统曲线 + 目标进程曲线叠加，无 TOP10 |
| GPU 使用率 | 左侧主曲线图 | 系统曲线，有 TOP10 Tooltip |
| 内存使用 | 右侧次要指标 | 数值 + 迷你曲线，无 TOP10 |
| CPU 温度 | 右侧次要指标 | 数值 + 迷你曲线 |
| 功耗 | 右侧次要指标 | 数值 + 迷你曲线 |
| 上传/下载速度 | 右侧次要指标 | 数值 + 迷你曲线 |

#### 进程级别指标（仅 CPU 和 GPU）
- 指定进程的 CPU、内存、提交内存占用
- TOP10 进程占用排行（CPU/内存排序）
- 同进程名多实例：曲线显示总占用，Tooltip 显示各子进程详情

## 二、页面布局设计

### 2.1 整体结构

```
┌─────────────────────────────────────────────────────────────────────┐
│ 顶部控制栏                                                          │
│ [设备选择] [开始/停止采集] [采集状态] [标记版本] [历史采集]            │
├─────────────────────────────────────────────────────────────────────┤
│ 时间轴选择器                                                        │
│ [30分钟] [60分钟] [全部] [时间轴滑块] [自定义时间输入]                │
│ [已标记版本列表 - 点击跳转]                                          │
├─────────────────────────────────────────────────────────────────────┤
│ 左侧主曲线图区 (65%)          │ 右侧栏 (35%)                         │
│ ├─ CPU 使用率曲线图           │ ├─ 次要指标卡片                      │
│ ├─ 提交内存曲线图             │ │   内存使用 / CPU温度 / 功耗        │
│ ├─ GPU 使用率曲线图           │ │   上传速度 / 下载速度               │
│                               │ ├─ CPU TOP10 迷你趋势线              │
│                               │ ├─ GPU TOP10 迷你趋势线              │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 顶部控制栏

#### 设备选择
- 下拉选择目标 Windows 设备
- 显示设备名 + IP

#### 采集按钮状态
| 状态 | 开始按钮 | 停止按钮 | 状态显示 |
|------|----------|----------|----------|
| 未采集 | 绿色可用 | 灰色禁用 | 无 |
| 正在采集 | 灰色禁用 | 红色可用 | 绿色闪烁 + 进程数 + 频率 |

#### 采集状态显示（正在采集时）
- 显示内容：`正在采集 | 频率: 5秒 | 目标: chrome.exe, node.exe 等 5 个进程`
- 省略规则：进程数 > 3 时省略显示
- Tooltip：鼠标悬停显示完整进程列表（进程名 + PID + 当前 CPU）

#### 版本标记入口
- **标记版本按钮**：将当前选中时间区间保存为对比版本
- **历史采集按钮**：查看该设备的历史采集记录

### 2.3 开始采集弹窗

点击"开始采集"弹出配置窗口：

```
┌──────────────────────────────────────────┐
│ 采集配置                              ✕  │
├──────────────────────────────────────────┤
│ 目标设备: Windows-001 (192.168.1.100)    │
│                                          │
│ 采集频率: [5] 秒  [1秒][5秒][10秒][30秒] │
│                                          │
│ 目标进程:                                │
│ ┌─────────────────────────────────────┐ │
│ │ 预设常用: chrome node python vscode │ │
│ │ [搜索框] [查询] [刷新列表]           │ │
│ │ ☑ chrome.exe PID:1234 CPU:5.2%     │ │
│ │ ☑ chrome.exe PID:2345 CPU:4.8%     │ │
│ │ ☐ chrome.exe PID:3456 CPU:2.5%     │ │
│ │ ☑ node.exe    PID:4567 CPU:6.1%    │ │
│ │ ☑ python.exe  PID:6789 CPU:3.2%    │ │
│ │ ...                                 │ │
│ └─────────────────────────────────────┘ │
│ 已选择: chrome(2) node(1) python(1) 清空 │
├──────────────────────────────────────────┤
│              [取消] [开始采集]            │
└──────────────────────────────────────────┘
```

#### 功能要点
- **采集频率**：输入框 + 快捷按钮，默认 5 秒
- **目标进程**：
  - 搜索框：模糊查询进程名
  - 预设常用：点击自动勾选该进程名下所有实例
  - 进程列表：显示进程名 + PID + 当前 CPU 占用
  - 多选：可勾选多个进程实例
- **停止采集**：直接停止，无需弹窗确认

### 2.4 时间轴选择器

#### 快速时间按钮
- 30分钟 / 60分钟 / 全部（去掉 1分钟、5分钟）

#### 时间轴滑块
- 可拖动窗口选择时间段
- 拖动窗口边缘调整起止时间
- 显示已标记版本区间（彩色标记）

#### 自定义时间输入
- 输入起止时间精确选择

#### 版本跳转
- 点击时间轴上的版本标记 → 自动跳转并选中该区间
- 点击下方版本列表 → 同样跳转

### 2.5 左侧曲线图区

#### CPU 使用率曲线图
- 系统曲线（蓝色）+ 目标进程曲线叠加
- 目标进程：不同进程用不同颜色区分
- Tooltip：
  - 时间点数值
  - 系统 CPU 占用
  - 各目标进程占用（显示子进程详情）
  - TOP10 进程排行

#### 提交内存曲线图
- 系统曲线 + 目标进程曲线叠加
- Tooltip：只显示数值，无 TOP10

#### GPU 使用率曲线图
- 系统曲线
- Tooltip：显示 TOP10 进程排行

### 2.6 右侧栏

#### 次要指标卡片
每个指标显示：当前数值 + 迷尼历史曲线
- 内存使用（占据两列宽度）
- CPU 温度
- 功耗
- 上传速度
- 下载速度

#### TOP10 进程概览区（迷你趋势线样式）
- TOP3 进程：显示迷你趋势线 + 进程名 + 占用率
- 其他进程：列表显示进程名 + 占用率
- 分为 CPU TOP10 和 GPU TOP10 两个区域

## 三、版本对比功能

### 3.1 版本标记
- 在时间轴上选择区间，点击"标记版本"
- 输入版本名称（如"启动阶段 baseline"、"优化后 v2"）
- 版本在时间轴上用彩色标记显示
- 版本默认包含全量数据（所有指标）

### 3.2 版本对比页面（子菜单入口）

#### 版本选择
- 显示所有已标记版本列表
- 用户勾选两个版本进行对比

#### 对比筛选
- **指标筛选**：可选择只对比某些指标（CPU/内存/GPU/功耗/温度等）
- **进程筛选**：可选择只对比某些目标进程

#### 对比结果
- **曲线叠加图**：两个版本的曲线在同一图上叠加显示
- **数据摘要表**：
  - 均值对比
  - 峰值对比
  - 差异百分比

## 四、数据存储策略

### 定期清理
- 设置保留周期（如保留最近 7天/30天/90天）
- 超出后自动清理

### 手动清理
- 支持手动删除采集记录

### 保护标记
- 用户可标记某些数据为"不清理"
- 标记的数据不会被自动清理

## 五、与 Worker 的交互

### 数据上报方式
- HTTP API 定时上报（用户可配置频率，默认 5 秒）

### 交互流程
1. 用户点击"开始采集" → 弹窗配置进程和频率
2. Worker 开始定时上报性能数据
3. 前端实时更新曲线图
4. 数据持久化存储，支持历史追溯
5. 用户点击"停止采集" → 结束采集

### 进程列表获取
- 从 Worker 查询当前运行进程列表
- 显示进程名 + PID + 当前 CPU 占用
- 支持模糊搜索和预设常用

## 六、技术要点

### 前端技术
- 图表库：ECharts（项目已有）
- 实时更新：WebSocket 或 HTTP 定时请求
- 时间轴组件：自定义实现或基于 ECharts dataZoom

### 后端技术
- 模块结构：参考现有 env_machine 模块
- 数据存储：PostgreSQL（按时间分区表）
- API 路由：`/api/core/performance-monitor`

### 数据模型
```
PerformanceRecord:
  - id: UUID
  - device_id: 外键
  - collect_time: 时间戳
  - cpu_usage: float
  - gpu_usage: float
  - commit_memory: float
  - memory_usage: float
  - cpu_temp: float
  - power: float
  - upload_speed: float
  - download_speed: float
  - top_processes: JSON (TOP10 数据)
  - target_processes: JSON (目标进程数据)

PerformanceVersion:
  - id: UUID
  - record_id: 外键
  - name: 版本名称
  - start_time: 开始时间
  - end_time: 结束时间
  - is_protected: 是否保护标记
```
