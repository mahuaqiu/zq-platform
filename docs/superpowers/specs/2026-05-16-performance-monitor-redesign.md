# 性能监控页面重新设计规格文档

> 创建日期: 2026-05-16
> 作者: Claude Code

## 1. 概述

### 1.1 背景
现有的性能监控页面设计不够理想，需要重新设计以提升用户体验和数据可视化效果。

### 1.2 目标
- 简化页面布局，突出核心数据
- 提供直观的指标切换方式
- 实现图表与详情面板的实时联动
- 设计更美观的时间导航条

### 1.3 范围
- 前端页面组件重构
- 交互逻辑优化
- UI 设计改进

## 2. 页面布局设计

### 2.1 整体布局结构

页面采用单图表主导型布局，分为三个主要区域：

```
┌─────────────────────────────────────────────────────────┐
│ [指标选择卡片区域]                                         │
├─────────────────────────────────────────────────────────┤
│ [时间导航条]                                              │
├─────────────────────────────────────────────────────────┤
│ [主图表区域 - 折线图]                                      │
├─────────────────────────────────────────────────────────┤
│ [目标进程明细 500px] │ [系统TOP10进程 500px]              │
└─────────────────────────────────────────────────────────┘
```

### 2.2 指标选择卡片

**位置**: 页面顶部

**组成**:
- 4个固定显示的指标卡片: CPU、GPU、内存、提交内存
- 1个"更多指标 ▼"按钮，点击弹出搜索选择弹窗

**样式**:
- 选中的指标: 蓝色背景（#409eff），白色文字
- 未选中的指标: 白色背景，蓝色边框，蓝色文字
- 圆角: 6px
- 内边距: 10px 20px
- 字体: 14px，font-weight: 600

**交互**:
- 点击卡片切换当前显示的指标
- 点击"更多指标"弹出选择弹窗

**更多指标弹窗设计**:
- 标题: "选择指标" + 关闭按钮（×）
- 搜索框: placeholder "搜索指标名称..."
- 最近搜索: 显示最近搜索过的3个指标（彩色标签风格）
- 搜索结果列表:
  - 左边: 中文名（如"CPU核心0温度"），灰色显示"未知传感器"表示未标记
  - 右边: 英文KEY（如"cpu_core_0_temp"，灰色小字）
  - 不显示数值
  - 选中项高亮背景 #f5f5f5
  - 最大高度 180px，超出滚动
- 弹窗宽度: 350px
- 弹窗位置: 相对于"更多指标"按钮定位

### 2.3 时间导航条

**位置**: 指标选择卡片下方

**组成**:
- 左侧快速选择按钮: 全部、5分钟、30分钟、60分钟
- 右侧可拖动导航条

**样式**:
- 背景条: 灰色（#e8e8e8），高度 6px，圆角 3px
- 选中区域: 蓝色（#409eff）
- 把手: 白色背景，蓝色边框（2px solid #409eff），宽 14px，高 16px，圆角 4px

**时间标签显示规则**:
- 把手距离 >= 10%: 两个时间标签分开显示，分别位于把手上方
- 把手距离 < 10%: 合并显示一个时间范围标签，放到导航条最上方，不遮挡把手
- 时间格式:
  - 分开显示: `YYYY-MM-DD HH:MM:SS`
  - 合并显示: `YYYY-MM-DD HH:MM:SS ~ HH:MM:SS`（第二个时间省略年月日）
- 合并标签样式: 白色背景，padding: 2px 8px，border-radius: 3px，轻微阴影
- 导航条区域高度:
  - 正常模式: 30px
  - 合并模式: 60px（为标签预留更多空间）

**交互**:
- 点击快速按钮自动调整时间范围
- 拖动把手精确选择时间范围
- 拖动过程中时间标签实时更新

### 2.4 主图表区域

**位置**: 时间导航条下方

**默认显示**: CPU使用率折线图

**图表类型**: ECharts 折线图

**数据系列** (按指标类型):
- CPU: 系统CPU（蓝色 #409eff）、进程CPU（绿色 #67c23a）
- GPU: 系统GPU（蓝色 #409eff）、进程GPU（绿色 #67c23a）
- 内存: 进程内存（灰色 #909399）
- 提交内存: 进程提交内存（紫色 #9b59b6）

**标题**: 根据当前指标显示，如"CPU使用率"、"进程内存"等

**图表高度**: 约 310px-320px（预留足够空间展示数据）

**MiniTooltip 样式**:
- 背景: 白色
- 圆角: 12px
- 阴影: `box-shadow: 0 4px 14px rgba(0, 0, 0, 0.2)`
- 内边距: 12px
- 时间显示: `YYYY-MM-DD HH:MM:SS`
- 数据显示: 颜色圆点 + 系列名称 + 数值

**交互**:
- 鼠标悬停时 MiniTooltip 实时显示当前时间点数据
- 鼠标悬停同步更新下方目标进程明细和TOP10面板数据
- 无悬停时显示最新数据

### 2.5 目标进程明细面板

**位置**: 主图表下方左侧

**宽度**: 固定 500px

**标题**: "目标进程明细"（不带指标类型后缀）

**时间显示**: 标题右侧显示当前时间 `YYYY-MM-DD HH:MM:SS`

**内容**:
- 子进程详情列表
- 每行显示: 进程名 + PID + 当前指标值
- 最后一行显示子进程实例总数

**样式**:
- 背景: 白色
- 圆角: 8px
- 内边距: 16px
- 列表项背景: #f9f9f9
- 列表项内边距: 6px 8px
- 字体: 11px

**显示规则**:
- CPU/GPU/内存/提交内存 都显示此面板
- 数值根据当前指标显示对应字段（cpu/gpu/memory/committed_memory）

### 2.6 系统TOP10进程面板

**位置**: 主图表下方右侧

**宽度**: 固定 500px

**标题**: "系统TOP10进程"

**时间显示**: 标题右侧显示当前时间

**内容**:
- 显示完整的10个进程排名
- 每行显示: 排名数字 + 进度条 + 进程名 + 数值
- 排名1-4使用彩色（蓝、绿、橙、红）
- 排名5-10使用灰色

**样式**:
- 进度条背景: #f5f5f5
- 进度条高度: 6px
- 圆角: 2px
- 字体: 11px

**显示规则**:
- 仅 CPU 和 GPU 指标显示此面板
- 内存、提交内存、其他指标不显示此面板

## 3. 数据联动逻辑

### 3.1 悬停联动
- 鼠标在图表上悬停时，MiniTooltip、目标进程明细、TOP10面板同步显示悬停时间点的数据
- MiniTooltip 显示绝对时间和数据值
- 目标进程明细显示该时间点的子进程详情
- TOP10显示该时间点的排名数据

### 3.2 无悬停状态
- 鼠标离开图表后，所有面板显示最新数据（时间序列中最后一个数据点）

### 3.3 指标切换
- 点击指标卡片切换当前指标
- 图表标题、数据系列、目标进程明细数值、TOP10数值同步切换
- TOP10面板仅 CPU/GPU 显示，其他指标隐藏

## 4. 数据结构

### 4.1 已有数据结构（保持不变）

**PerformanceData**:
```typescript
interface PerformanceData {
  target_processes: ProcessData[];
  top10_cpu: ProcessInstance[];
  top10_gpu: ProcessInstance[];
  // 其他字段...
}
```

**ProcessData**:
```typescript
interface ProcessData {
  process_name: string;
  instances: ProcessInstance[];
}
```

**ProcessInstance**:
```typescript
interface ProcessInstance {
  pid: number;
  cpu: number;
  memory: number;
  gpu: number;
  committed_memory: number;
}
```

### 4.2 图表数据系列

**计算属性**:
- `cpuChartSeries`: [{ name: '系统', data: [...] }, { name: '进程', data: [...] }]
- `gpuChartSeries`: [{ name: '系统', data: [...] }, { name: '进程', data: [...] }]
- `memoryChartSeries`: [{ name: '进程内存', data: [...] }]
- `commitMemoryChartSeries`: [{ name: '进程提交内存', data: [...] }]

## 5. 组件设计

### 5.1 主页面组件 (index.vue)

**职责**:
- 管理当前选中指标状态
- 管理时间范围状态
- 管理悬停时间点状态
- 组合各子组件

**状态**:
- `currentMetric`: 当前选中指标
- `timeRange`: 时间导航条选中范围
- `hoverTimestamp`: 悬停时间点（null 表示无悬停）

### 5.2 MiniTooltip 组件

**职责**: 显示悬停时间点的数据

**Props**:
- `timestamp`: 时间戳
- `seriesData`: 系列数据数组 [{ color, name, value }]

**样式**: 保持现有白色背景、圆角12px、阴影风格

### 5.3 ChartPanel 组件

**职责**: 渲染折线图，处理悬停事件

**Props**:
- `metric`: 当前指标
- `chartSeries`: 图表数据系列
- `timeRange`: 时间范围

**Events**:
- `hover`: emit 悬停时间点
- `leave`: emit 离开事件

### 5.4 ProcessDetailPanel 组件

**职责**: 显示目标进程明细

**Props**:
- `metric`: 当前指标
- `processData`: 进程数据
- `timestamp`: 当前时间点

**宽度**: 固定 500px

### 5.5 Top10Panel 组件

**职责**: 显示系统TOP10进程

**Props**:
- `metric`: 当前指标
- `top10Data`: TOP10数据
- `timestamp`: 当前时间点

**宽度**: 固定 500px

**显示条件**: 仅 currentMetric 为 'cpu' 或 'gpu' 时显示

### 5.6 MetricSelector 组件

**职责**: 指标选择卡片

**Props**:
- `currentMetric`: 当前选中指标

**Events**:
- `change`: emit 切换指标名称

### 5.7 TimeNavigator 组件

**职责**: 时间导航条

**Props**:
- `totalDuration`: 数据总时长
- `quickRanges`: 快速选择按钮配置

**Events**:
- `rangeChange`: emit 时间范围变化 { start, end }

**时间标签逻辑**:
```javascript
// 计算把手距离百分比
const handleDistance = (rightPosition - leftPosition) / 100;

if (handleDistance >= 0.1) {
  // 分开显示两个时间标签
  showLeftLabel = true;
  showRightLabel = true;
  showCombinedLabel = false;
} else {
  // 合并显示时间范围
  showLeftLabel = false;
  showRightLabel = false;
  showCombinedLabel = true;
}
```

## 6. 实现要点

### 6.1 悬停同步
- 使用 ECharts 的 `mousemove` 事件获取悬停位置
- 计算对应时间点
- 通过 props 传递到各面板组件
- 面板组件根据时间点查找对应数据

### 6.2 指标切换
- 点击卡片更新 `currentMetric` 状态
- 所有依赖 `currentMetric` 的组件自动更新
- TOP10面板根据 `currentMetric` 决定是否渲染

### 6.3 时间导航
- 快速按钮点击设置固定范围
- 把手拖动实时计算百分比转时间
- 时间标签动态切换显示模式

### 6.4 响应式设计
- 图表区域自适应宽度
- 底部两个面板固定 500px 宽度
- 小屏幕时考虑滚动或折叠

## 7. 移除内容

- 移除原有高级指标展示区域（hwinfo_raw 200+传感器专用UI区域）
- 移除原有时间选择器
- 移除 TOP10 内的 CPU/GPU 切换标签（由顶部指标卡片统一控制）

**说明**: hwinfo_raw 传感器数据源保留，可通过"更多指标"弹窗搜索访问。需要维护一个中文名称映射表，将英文KEY映射到中文显示名。

## 8. 成功标准

- 页面布局清晰，一眼可见核心数据
- 指标切换流畅，数据同步正确
- 悬停联动实时响应
- 时间导航条美观易用
- 所有10个TOP进程完整显示
- 时间标签不重叠
